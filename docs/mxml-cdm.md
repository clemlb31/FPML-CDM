# MXML ↔ CDM Component

Bidirectional conversion between **MXML** (Murex XML) and **FINOS CDM 6.19.0**.

| Direction | Status | Strategy |
|---|---|---|
| **MXML → CDM** | 🔗 By chaining | **MXML → FpML → CDM** — implemented via `io.fpmlcdm.mxml.cdm.MxmlToCdmConverter` (reuses [mxml-fpml](mxml-fpml.md) then [fpml-cdm](fpml-cdm.md)) |
| **CDM → MXML** | ⏳ Future | **CDM → FpML → MXML** (symmetric) |

## Why chaining first

The **FpML→CDM pipeline is mature (530/530)**. As soon as **MXML→FpML** produces valid FpML, we get
MXML→CDM for free by chaining, without rewriting the CDM logic. This is the fastest and least risky
path to a working MXML→CDM.

```
MXML  ──(mxml.fpml)──►  FpML  ──(FpmlToCdmConverter)──►  CDM TradeState
```

### Validation
Each [data/ground_truth/mxml-fpml/](../data/ground_truth/mxml-fpml/)`OUT_*` folder provides the **expected CDM** (`*_CDM.json`). The end-to-end
test compares the CDM obtained by chaining against the `*_CDM.json` via the existing `SemanticDiff` — exactly
the same mechanics as [fpml-cdm](fpml-cdm.md).

## Future option: direct MXML→CDM mappers
If chaining loses information (the intermediate FpML does not express all of the MXML), we could write direct
MXML→CDM mappers reusing `io.fpmlcdm.fpml.cdm.common/*` and the CDM builders — at the cost of duplicating the product
logic. Only worth considering if a concrete case justifies it.

## Prerequisites
- MXML→FpML operational (see [mxml-fpml.md](mxml-fpml.md)).
- Resolvable CDM build (Murex Nexus or `mvn -s` Central) — cf. [schemas-and-validation.md](schemas-and-validation.md).

## Dataset
`data/generated/mxml-cdm/<family-subfamily>/{mxml,cdm,ignored}/` — **239 pairs** (MXML input + expected CDM + `ignored`
mask), generated by chaining and **byte-identical** to the reference CDM. Construction:
[reports/mxml-cdm-dataset-239.md](../reports/mxml-cdm-dataset-239.md). 106 lifecycle events remain
excluded (empty CDM: FpML→CDM does not model `BusinessEvent`).

## Known fidelity (LLM-judge audit, 2026-06-24)
Semantic audit of the 239 pairs (LLM-as-judge + adversarial pass):
[reports/mxml-cdm-llm-judge.md](../reports/mxml-cdm-llm-judge.md).

- **82 % faithful** (PASS+MINOR); vanilla core (IRS/CS/CDS/OIS) well translated, %→decimal conversion correct.
- **37 confirmed material defects** (15.5 %, after adversarial rebuttal of the 6 false positives).
- Where info is lost: mostly at the **FpML→CDM** stage (discarded envelopes: optionality, inflation, premiums,
  known-amount); the **MXML→FpML** stage introduces index distortions (RFR→LIBOR) and counterparty
  collapses. ⚠️ "byte-identical vs reference" proves reproducibility, **not** fidelity.
